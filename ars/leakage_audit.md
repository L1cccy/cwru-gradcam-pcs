# 泄漏审计 — 方向五：Grad-CAM 物理一致性量化

## 审计范围

本审计检查 CWRU 12kHz DE 数据在训练/测试划分过程中可能引入的 5 类数据泄漏路径。

## 1. 文件级完整性

**检查项**: 是否有文件包含多个故障类别（交叉污染风险）

| 检查 | 结果 | 证据 |
|------|------|------|
| 每个 .mat 文件是否仅含一种故障类型？ | ✅ PASS | CWRU 文件按 B/IR/OR 分目录，每文件对应一次采集 |
| 同一目录内的文件是否来自不同采集？ | ✅ PASS | 按故障尺寸 + 负载分层，每文件独立采集 |
| 是否有重叠数据（同一段信号出现在多个文件）？ | ✅ PASS | 无已知重叠 |

**评级**: 🟢 低风险

## 2. recording_id 划分检查

**检查项**: 按 recording_id 分组的训练/验证/测试能否保证独立性

| 检查 | 结果 |
|------|------|
| recording_id 是否唯一标识一次独立采集？ | ✅ 是 |
| 不同 recording_id 之间是否存在信号重叠？ | ✅ 否（独立采集） |
| 同一轴承在多次采集（如 007_0, 007_1）上是否用了相同的故障轴承？ | ⚠️ 同一物理轴承可能在不同负载下重复采集（同一故障，不同转速/负载） |
| 这是泄漏吗？ | **不是**——不同 recording_id 虽然可能来自同一物理轴承，但是不同采集，信号内容（转速、负载）不同。如果模型在训练时看到 007_0（1797rpm），测试时看到 007_1（1772rpm），这实际上是跨工况泛化，是合法的评估。关键是确保同一段精确信号不会同时出现在 train 和 test 中，这个条件通过 recording_id 划分已经满足。 |

**评级**: 🟢 低风险

## 3. 相邻窗口泄漏

**检查项**: 滑动窗口切分后，相邻窗口是否有信息重叠？

| 检查 | 结果 |
|------|------|
| 窗口是否重叠？ | ⚠️ 是（hop_length=512 < window_length=1024，50% 重叠） |
| 重叠窗口是否可能出现在不同划分中？ | **❌ 不会**——所有窗口保留源 recording_id，同一 recording_id 的所有窗口只在一个划分中 |
| 相邻窗口是否高度相似？ | ⚠️ 是——这是本文要研究的现象之一（用 Grad-CAM 评估跨窗口稳定性） |

**评级**: 🟢 低风险——通过 recording_id 划分已控制
**透明声明**: 相邻窗口在时域上有 50% 重叠，但通过 recording_id 分组确保不会泄漏到测试集

## 4. 归一化泄漏

**检查项**: 归一化统计量是否在训练/测试集独立计算？

| 检查 | 方案 | 风险 |
|------|------|------|
| 标准化（z-score） | 在训练集上计算 μ_train, σ_train → 对训练和测试集统一应用 | 🟢 无泄漏 |
| 如果先归一化再划分 | ❌ 禁止——测试集信息污染训练统计量 | 🔴 高危（需在代码中确保） |

**评级**: 🟡 中风险——需要代码层面确保先划分再归一化
**实施要求**: 代码中必须先按 recording_id 划分，再对训练部分计算 μ, σ

## 5. 增强泄漏

**检查项**: 增强操作是否在划分前执行？

| 检查 | 方案 |
|------|------|
| 噪声增强 | 在划分后对训练集执行（本文的噪声测试仅在推理时注入干净训练模型的测试集，不涉及训练时增强） |

**评级**: 🟢 无风险——本文不涉及训练时数据增强

## 6. 故障频率计算泄漏

**检查项**: 故障频率的计算是否依赖测试集信息？

| 检查 | 方案 | 风险 |
|------|------|------|
| BPFO/BPFI/BSF 计算 | 基于轴承几何参数和标称转速，纯物理公式 | 🟢 无泄漏 |
| 频带宽度（±Δf）选择 | 基于 STFT 频率分辨率（23.44 Hz），非数据驱动 | 🟢 无泄漏（可在训练集上设定） |

**评级**: 🟢 无风险

## 7. 负对照设计中的泄漏

**检查项**: 频带平移负对照是否使用测试集信息确定平移量？

| 检查 | 方案 | 风险 |
|------|------|------|
| 平移量确定 | 在实验设计阶段随机固定（伪随机种子确定），不依赖数据 | 🟢 无泄漏 |

**评级**: 🟢 无风险

## 总体安全性评级

| 泄漏路径 | 评级 | 控制措施 |
|---------|------|---------|
| 文件交叉污染 | 🟢 低 | 自然满足 |
| recording_id 重叠 | 🟢 低 | 严格按 id 分组 |
| 相邻窗口 | 🟢 低 | recording_id 分组确保不泄漏 |
| 归一化 | 🟡 中 | 代码中确保先划分再归一化 |
| 数据增强 | 🟢 无 | 不涉及训练时增强 |
| 故障频率 | 🟢 无 | 纯物理公式 |
| 负对照平移量 | 🟢 无 | 随机种子固定 |

**总体判定**: ✅ PASS — 泄漏风险总体可控。唯一需要注意代码层面的归一化顺序。准予进入实验阶段。
